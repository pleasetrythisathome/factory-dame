# Feedback: light structure is welcome on factory-dame — the earlier memory overstated it

## The correction

Earlier today (memory `2026-04-16-1-feedback-skip-ticketing-on-side-projects`) I recorded Scarlet's pushback on ticketing as a near-absolute rule: "Do NOT propose creating Backlog.md tickets…unless Scarlet asks for one explicitly."

In the next session (same day), Scarlet corrected the framing:

> "that memory about not doing backlog was way overplayed. this is just a fun project and i won't want to create a ton of structure but some is important and helpful sometimes, like the ticket for the handoff"

## What the real stance is

- **Light touch, not zero touch.** Scarlet doesn't want a ticketed workflow driving the project, but she doesn't object to structure when it earns its keep.
- **Concrete example she endorsed:** the handoff doc at `backlog/docs/neurodynamics-handoff-2026-04-16.md` and the 10 backlog tickets (`task-001` … `task-010`) created in the prior session. She chose to keep them rather than rebase them out, after seeing the contradiction with the older memory.
- **The spirit the earlier memory got right:** this is a creative counterweight to aswritten. Velocity tracking and reflexive ticketing of every future idea would import the wrong energy.
- **The spirit it got wrong:** treating "no process" as a rule. A handoff doc, an occasional ticket, or a session summary are fine — and sometimes actively helpful — when they're carrying real weight.

## How to apply going forward

- **Default to lightweight.** If a thought occurs to you during the session, note it in conversation or in-session TaskCreate — don't reflexively open a Backlog.md ticket.
- **Structure is fine when it earns its keep.** Handoff docs between sessions, tickets for work that's actually on-deck, decision docs for non-trivial calls — all welcome. The test is "does this carry weight later?", not "is this Backlog.md?".
- **Still no ticket inflation.** Don't propose creating 18 tickets for every idea downstream. Propose one when it's warranted.
- **Session boundaries:** a handoff doc is now an approved form (it worked). Memories and handoff docs can coexist.
